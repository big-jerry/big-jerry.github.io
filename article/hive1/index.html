HIVE DDL 创建表的语句介绍

  • 创建列表：

例：创建emp表并制定分隔符和路径

  • 加载文件数据到列表：
    LOAD DATA LOCAL INPATH ‘/home/hadoop/data/emp.txt’ into table emp;
    PS：有LOCAL 是从本机Liunx文件系统导入，无LOCAL 默认从HDFS文件系统导入

PS:加载文件数据到列表如果显示为NULL，基本都是分隔符的问题引起。

  • 快速创建列表的方式：

    • 根据已经存在的列表创建新的列表：
      CREATE [TEMPORARY] [EXTERNAL] TABLE [IF NOT EXISTS] [db_name.]table_name
      LIKE existing_table_or_view_name
      [LOCATION hdfs_path];
    • 例：根据epm表创建emp1表，仅拷贝emp表的结构。
    • 例：根据epm表创建emp2表，并拷贝emp表的结构和数据。

  • 修改列表名称
    ALTER TABLE table_name RENAME TO new_table_name;
    例：修改emp2表名为emp_new。

  • 删除列表
    DROP TABLE [IF EXISTS] table_name [PURGE];
    例：删除emp_new表
    hive (hadoop)> drop table emp_new;

  • 清空列表数据
    TRUNCATE TABLE table_name [PARTITION partition_spec];
    例：清空emp1列表的数据
    hive (hadoop)> truncate table emp1;

修改外部和内部表

ALTER TABLE table_name SET TBLPROPERTIES table_properties;
table_properties:<font></font>

(property_name = property_value, property_name = property_value, … )
例：把emp2内部表修改成外部表

外部表和内部表的区别

* 内部表删除时: HDFS 和元数据META 都被删除
* 外部表删除时: HDFS不删除 ，仅元数据被删除

创建分区表

CREATE TABLE invites (foo INT, bar STRING) PARTITIONED BY (ds STRING);
例：创建一个分区表


`PS：分区列并不是一个“真正的”表字段，其实是HDFS上表对应的文件夹下的一个文件夹。`
分区表加载数据一定要指定分区字段,对于分区表操作，如果你的数据是写入HDFS

默认sql是查询不到的.
例：创建order_partition2分区表，并写入数据到该文件夹,查看分区表数据。

解决办法：

MSCK [PEPAIR] TABLE table_name [ADD/DROP/SYNC [PARTITIONS];

ALTER TABLE table_name ADD [IF NOT EXISTS] PARTITION partition_spec [LOCATION 'location'][, PARTITION partition_spec [LOCATION 'location'], ...];
partition_spec:
  : (partition_column = partition_col_value, partition_column = partition_col_value, ...)
hive (hadoop)> msck repair table order_partition2;
OK

PS：使用分区表时，加载数据一定要指定我们的所有分区字段
